Problem Solving and Overcoming Roadblocks

Stuck on a Simple Problem

We often think that the path to mastering a skill is a straight line, but sometimes it’s more like a winding road with unexpected detours. One day, while tackling one of our Javascript Kata exercises (little challenges designed to hone our JS skills), I found myself caught on a simple problem.

We had to check if a number was divisible by 3, 5, both, or neither, and return a specific word based on the result. It was straightforward enough for a single number, but the challenge was to extend this logic to an array of numbers. I started with confidence, writing a function to solve the problem for one number. But then, faced with an array of numbers, my brain hit a roadblock.

That’s when I turned to a technique that I’ve since learnt to love: writing out pseudocode. Sketching out the steps in plain language quickly cleared the fog, and I could see the solution taking shape. Piece by piece, I implemented the steps, feeling satisfaction as the function began to work perfectly for the array.

Solving Problems Elegantly

In another exercise, we were asked to add a list of students to a gradebook. The instructions made it sound like we should manually link each student to the gradebook, but I saw an opportunity to write a function. I wanted to practice using the for…of loop, which is so clean and readable. So, I dove in, writing the function from memory and using console.log to check my progress. It felt great seeing the code come together; I used ChatGPT to catch a few minor syntax errors and I was done. It was a moment of pure confidence and competence.

Reflecting on Problem-Solving Techniques

Reflecting on these experiences, I realize how crucial various problem-solving techniques have become:

  • Pseudocode: This has quickly become one of my favorite methods. It’s like laying down a roadmap before embarking on a journey, making the actual coding part much smoother. Plus, it doubles as great comments for future reference.
  • Trying Something: Sometimes, just diving in and trying to implement a solution, even if it’s not perfect at first, is the best way to learn and improve.
  • Rubber Ducky Method: Explaining my code to ChatGPT, much like talking to a rubber duck, helps clarify my thoughts and spot errors I might have missed.
  • Reading Error Messages: This has become less daunting with practice. The more I encounter errors, the better I get at understanding and resolving them.
  • Console.logging: An indispensable tool for pinpointing exactly where things go wrong. It’s like having a window for my code.
  • Googling: The classic go-to move. There’s a wealth of knowledge out there, and knowing how to search for it effectively is key.
  • Asking for Help: Whether from peers or coaches, I’m pretty comfortable reaching out when I hit a wall.
  • Reflection: Constantly looking back at what worked and what didn’t helps refine my approach and improve over time.

The Importance of Asking for Help

As for asking for help, I’ve never been reluctant. I know I don’t have all the answers, and I’m always ready to learn from others. The only time I hesitate is when I’m pushing myself to excel and don’t want to come across as too boastful (a nod to NZ’s tall poppy syndrome). But generally, I embrace the collaborative spirit of learning.

Problem-solving in coding isn’t just about writing the correct code—it’s about the journey, the techniques you develop, and the mindset you cultivate. Each challenge is a stepping stone, each roadblock a chance to learn, and every solution a reason to celebrate.
