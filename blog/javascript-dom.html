JavaScript and the DOM: A Beginner's Guide

JavaScript is the engine that brings web pages to life, adding functionality and interactivity to the static structures created with HTML and CSS. Today, I'm exploring some fundamental concepts in JavaScript, including control flow and loops, how to interact with the DOM (Document Object Model), and the differences between arrays and objects. I'll also touch on the power of functions to make your code more efficient and reusable. Whether you're just starting out or looking to solidify your understanding, this guide aims to make these core ideas approachable and relatable.

JavaScript and its Relationship to HTML and CSS

If HTML is the structure, and CSS is the styling; then, JavaScript is the logic — it makes everything work. Want a quote to be generated when you click a button? JavaScript. Want to pull data on page load? JavaScript. It’s the dynamic element that brings your web pages to life, handling everything from simple tasks to complex interactions. By integrating JavaScript with HTML and CSS, you can build responsive, engaging, and user-friendly web applications.

Control Flow and Loops

From my background in photography, loops remind me a lot of editing a big batch of photos; load photo, +5 temp, -2 exposure, next photo. Repeat until all the photos are edited. In programming, control flow and loops allow us to repeat actions and make decisions in our code.

The DOM and How You Interact with It

The DOM (Document Object Model) is a collection (similar to an array) of all the elements on a webpage. It’s a giant tree of nodes, every paragraph, heading, button, and div on the page. You can interact with it in JavaScript by calling the document object, like document.querySelector.

Accessing Data from Arrays vs. Objects

Arrays and objects have a lot of similarities; most significantly, they are both used to store multiple values in one property.

  • An array is pretty simple; it is a list of values grouped with square brackets and separated with commas. e.g., people = ['hemi', 'joseph', 'gerard'].

    Each item has an index; a secret number assigned to it based on its order in the array — a bit of a quirk, these numbers start from 0. e.g., in the people array from before, 'hemi' is 0, 'joseph' is 1, etc. You can get the item you want from the array by writing the array name and the index you want in square brackets e.g., people[0] would be 'hemi'.

  • An object is also a list of values, except each value is actually a key-value pair, which allows for easy storing and parsing of diverse information. e.g., 

    
                    hemi = { 
                      favoriteColor: 'blue', 
                      favoriteNumber: 12, 
                      isCool: true 
                    }

    You can access data by calling the key with dot notation e.g., hemi.favoriteNumber. Because each value has a key, instead of just an index number, it gives you a clue about the type of data you can expect to find; in this case, hemi.favoriteNumber is an integer with a value 12.

What Are Functions and Why Are They Helpful

Functions are a way to bundle and compartmentalize an action or series of actions. This is incredibly useful because it makes it possible to define these actions and then refer back to them again and again in the future.

This is also a bit similar to editing photos. Let’s say you want to crop a photo, rotate it 90 degrees, and then export it to your desktop. You could do each of these steps for every photo individually, or you could bundle these actions together into a function called standardEdit ready to run on any photo you want.

Whether you add all these actions to one function, or whether you want to split them up into separate functions (maybe you don’t want to crop every photo), is your choice and will make sense to your specific situation.
