The Difference Between Margin, Border, and Padding

Today’s lesson dives into the box model in HTML and CSS. Think of HTML as a series of boxes (like <div> elements) that we style using CSS. Three key properties to know are margin, border, and padding.

Margin creates space outside the box—useful for separating elements like siblings and parent elements. Border defines the outer boundary of the box (it can be styled to create a visible line or be invisible with a width of 0 or a transparent color). Padding creates space inside the box between the edge and its contents.

Here is a visual representation:

              +-------------------------------------------+
              |                  Margin                   |
              |  +-------------------------------------+  |
              |  |               Border                |  |
              |  |  +-------------------------------+  |  |
              |  |  |            Padding            |  |  |
              |  |  |  +-------------------------+  |  |  |
              |  |  |  |         Content         |  |  |  |
              |  |  |  +-------------------------+  |  |  |
              |  |  +-------------------------------+  |  |
              |  +-------------------------------------+  |
              +-------------------------------------------+

You can apply margin, border, and padding to all sides equally (e.g., padding: 10px) or target specific sides individually (e.g., padding-top: 2rem). There's also the combination option (e.g., padding: 1vw 5px 0 5px).

Understanding the box-sizing property is essential—by default, box-sizing: content-box means the width and height include only the content, not the padding and border. But setting box-sizing: border-box includes padding and border in the box’s total width and height (highly recommended for easier size control).

Finally, HTML was originally for text blocks (<h1>, <p>, etc.), leading to a quirk known as collapsing margins. When two block elements stack, their top and bottom margins overlap, and the resulting margin is the larger of the two, not their sum.

That’s today’s breakdown of the box model—keep experimenting with margin, border, and padding to see how they affect your layouts!
